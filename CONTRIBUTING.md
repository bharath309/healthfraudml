# Contributing to HealthFraudML

Thank you for your interest in contributing to HealthFraudML! This framework aims to bridge the gap between ML research and practical healthcare fraud detection.

## Areas Where Contributions Are Especially Valuable

- Additional ML model implementations
- Healthcare-specific feature engineering approaches
- Privacy-preserving techniques (federated learning, differential privacy)
- Benchmarking on additional datasets
- Documentation and tutorials
- Bug reports and fixes

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Write tests for your changes
4. Ensure all tests pass (`pytest tests/`)
5. Submit a pull request

## Code Style

- Follow PEP 8
- Include docstrings with references to relevant literature
- Add type hints to function signatures
